Modelado del sistema.

El modelado de sistemas es el proceso para desarrollar modelos abstractos de un sistema, donde cada modelo presenta una visión o perspectiva diferente de dicho sistema. Lenguaje de Modelado Unificado (UML). Los modelos se usan durante el proceso de ingeniería de requerimientos para ayudar a derivar los requerimientos de un sistema.

Es posible desarrollar modelos tanto del sistema existente como del sistema a diseñar:
  • Los modelos del sistema existente se usan durante la ingeniería de requerimientos.
  • Los modelos del sistema nuevo.

El aspecto más importante de un modelo del sistema es que deja fuera los detalles.

Usted puede desarrollar diferentes modelos para representar el sistema:
  • Una perspectiva externa, donde se modelen el contexto o entorno del sistema.
  • Una perspectiva de interacción, donde se modele la interacción entre un sistema y su entorno.
  • Una perspectiva estructural, donde se modelen la organización de un sistema o la estructura de datos.
  • Una perspectiva de comportamiento, donde se modele el comportamiento dinámico del sistema.
Usan diagramas definidos en UML.
  • Diagramas de actividad.
  • Diagramas de caso de uso.
  • Diagramas de secuencias.
  • Diagramas de clase.
  • Diagramas de estado.
Hay tres formas en que los modelos gráficos:
  • Como medio para facilitar la discusión sobre un sistema existente o propuesto.
  • Como una forma de documentar un sistema existente.
  • Como una descripción detallada del sistema que sirve para generar una implementación de sistema.

El Lenguaje de Modelado Unificado es un conjunto compuesto por 13 diferentes tipos de diagrama para modelar sistemas de software.

Modelos de interacción.

Todos los sistemas incluyen interacciones de algún tipo. Éstas son interacciones del usuario, que implican entradas y salidas del usuario.

Modelado de interacción:
  • Modelado de caso de uso, que se utiliza principalmente para modelar interacciones entre un sistema y actores externos (usuarios u otros sistemas).
  • Diagramas de secuencia, que se emplean para modelar interacciones entre componentes del sistema, aunque también pueden incluirse agentes externos.

Generalización.

La generalización es una técnica cotidiana que se usa para gestionar la complejidad. En vez de aprender las características detalladas de cada entidad que se experimenta, dichas entidades se colocan en clases más generales (animales, automóviles, casas, se aprenden las características de dichas clases. Esto permite deducir que diferentes miembros de estas clases tienen algunas características comunes En el modelado de sistemas, con frecuencia es útil examinar las clases en un sistema, con la finalidad de ver si hay ámbito para la generalización. La generalización se muestra como una flecha que apunta hacia la clase más general. En una generalización, los atributos y las operaciones asociados con las clases de nivel superior también se asocian con las clases de nivel inferior. parte de la jerarquía de generalización.

Ingeniería dirigida por modelo.

La ingeniería dirigida por modelo (MDE) es un enfoque al desarrollo de software donde los modelos, y no los programas, son las salidas principales.

Los principales argumentos a favor y en contra de MDE son:
  • En favor de la MDE La ingeniería basada en modelo permite a los ingenieros pen- sar sobre sistemas en un nivel de abstracción elevado.
  • Contra la MDE Como se analizó anteriormente, los modelos son una buena forma de facilitar las discusiones sobre un diseño de software

Arquitectura dirigida por modelo.

Transformaciones MDA,
El método de MDA recomienda la producción de tres tipos de modelo de sistema abstracto:
  • Un modelo independiente de computación (CIM) que modela las importantes abstracciones.
  • Un modelo independiente de plataforma (PIM) que modele la operación del sistema sin referencia a su implementación.
  • Modelos específicos de plataforma (PSM) que son transformaciones del modelo independiente de plataforma con un PSM separado para cada plataforma de aplicación.
